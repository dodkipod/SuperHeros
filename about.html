Hello


As mentioned on the main page, this project is about using rest api's and promises/async functions. It is meant to be viewed on a computer and is not responsive, as I didn't want to spend too long on design and layout. Requests to api are wrapped in a timeout function in order to better simulate the asynchronicity.

The purpose of this project was to get comfortable with both of the promise syntaxs (.then and async/await) and to practice the SOLID principles (where applicable). If you look at the index.html file you will see a fourth input in a comment. If you "un-comment" it you will see the site remains functional even with an additional input. The design may break a little (again, not the point of this project), but the code behind it remains functional.
